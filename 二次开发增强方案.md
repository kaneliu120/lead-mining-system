#### Plan ① Integrate StaffSpy → LinkedIn Hiring Intent Signal Mining

**Core Value**: LinkedIn hiring data is the strongest B2B buying intent signal — companies that are expanding have budgets and needs.

**Technical Approach**:

```python
# New file: app/miners/plugins/linkedin_miner.py
# Inherits existing BaseMiner interface, no changes to existing code needed

from staffspy import scrape_jobs
from app.miners.base import BaseMiner, MinerConfig, MinerHealth
from app.models.lead import LeadRaw, LeadSource

class LinkedInMiner(BaseMiner):
    """
    Scrapes LinkedIn job postings via StaffSpy,
    extracts employer information as a high-intent lead data source.
    Job count = company growth signal = high-quality leads.
    """

    @property
    def source_name(self) -> LeadSource:
        return LeadSource.LINKEDIN  # must be added to the LeadSource enum

    async def mine(self, keyword, location, limit):
        jobs = await asyncio.to_thread(
            scrape_jobs,
            site_name=["linkedin"],
            search_term=keyword,
            location=location,
            results_wanted=limit,
        )
        # Aggregate by company, deduplicate then yield LeadRaw
        # metadata stores: hiring_roles, job_count, seniority_mix
        for company, group in jobs.groupby("company"):
            yield LeadRaw(
                source=LeadSource.LINKEDIN,
                business_name=company,
                metadata={
                    "job_count": len(group),
                    "hiring_roles": group["title"].tolist()[:5],
                    "linkedin_url": group["company_url"].iloc[0],
                }
            )
```

**Registration** (one line of code, non-invasive):
```python
# Append to config.py
orchestrator.register(LinkedInMiner(config))
```

**Advantages**:
- Zero API cost (StaffSpy uses LinkedIn Voyager internal API)
- More hiring = higher priority leads
- Fully compatible with existing Orchestrator/deduplication/Fallback mechanism

**Risk**: LinkedIn ToS violation risk, recommend rate limiting (≤50 requests/hour).

#### Plan ② Integrate ScrapeGraphAI → Deep Company Website Intelligence

**Core Value**: Upgrades enrichment data from "Google Maps basic info" to "actual business intelligence from company websites", enabling highly personalized outreach emails.

**Technical Approach**:

```python
# Modify file: app/enrichers/gemini_enricher.py
# Add website intelligence collection step before the enrich() method

from scrapegraphai.graphs import SmartScraperGraph

async def _fetch_website_intel(self, website: str) -> dict:
    """Scrape key information from company website using ScrapeGraphAI"""
    if not website:
        return {}
    try:
        result = await asyncio.to_thread(
            SmartScraperGraph(
                prompt="Extract the company's main products/services, team size clues, recent developments, and technology tools or platforms in use",
                source=website,
                config={
                    "llm": {
                        "model": "google_genai/gemini-1.5-flash",
                        "api_key": self.api_key,
                    },
                    "headless": True,
                    "timeout": 30,
                }
            ).run
        )
        return result if isinstance(result, dict) else {"summary": str(result)}
    except Exception as e:
        logger.warning(f"Website intel fetch failed for {website}: {e}")
        return {}
```

**Integration with Gemini Enrichment**:
```
Original Prompt input: Google Maps basic data (name, address, rating)
Enhanced Prompt input: the above + website product details + tech stack + team info
                  ↓
outreach_angle quality:
  Before: "Hello, we offer digital solutions..."
  After: "I noticed your website currently uses WordPress, and your team is hiring for e-commerce operations..."
```

**Cost Estimate**: ~$0.0003/call (Gemini Flash call)

```python
async def validate_email(email: str) -> bool:

    Validate email deliverability.
```

#### Plan ④ Multi-Round Follow-up Sequence (3-touch Email Cadence)

**Core Value**: Single email reply rate ~5%; 3-round sequence reply rate can reach 15-20%.

**Database Changes** (minimal):

```sql
-- Add two fields to the existing outreach_log table
ALTER TABLE outreach_log ADD COLUMN sequence_step INT DEFAULT 1;
ALTER TABLE outreach_log ADD COLUMN next_followup_at TIMESTAMPTZ;
```

**Follow-up Sequence Design**:

```
Day 0:  Initial email (value proposition + specific entry point)
         ↓ 4 days no reply
Day 4:  Follow-up 1 (customer case studies + social proof)
         ↓ 4 days no reply
Day 8:  Follow-up 2 (resource sharing, lower decision barrier)
         ↓ 6 days no reply
Day 14: Break-up email ("Is now not a good time? Can we reconnect in 60 days?")
         ↓ No reply → mark INACTIVE
```

**n8n Workflow Modifications**:
- `workflow #03 (sales-outreach)` add sequence state decision node
- `workflow #04 (reply-detection)` interrupt sequence when reply is detected
- Add new scheduled workflow: daily scan of `next_followup_at < NOW()` to trigger follow-ups

**LangGraph Node Reuse**: The existing `GenerateEmailNode` accepts different `sequence_step` parameters to generate emails for each round, with no need to rewrite node logic.

#### Plan ⑥ ChromaDB Semantic Search API (Reuse existing vector index)

**Core Value**: ChromaDB already has data written but no query endpoint at all — activate existing capability at zero cost.

```python
# Add one endpoint to api.py (about 20 lines of code)

@app.get("/rag/similar")
async def semantic_search(
    query: str = Query(..., description="Natural language query"),
    limit: int = Query(10, le=50),
):
    """
    Search leads using vector semantic similarity.
    Examples:
      /rag/similar?query=fast-growing e-commerce companies
      /rag/similar?query=traditional retailers needing digital transformation
      /rag/similar?query=F&B chains similar in scale to Jollibee
    """
    results = state.chroma.query(query_text=query, n_results=limit)
    return {"results": results, "query": query}
```

#### Plan ⑦ Intelligent Reply Classification (Upgrade n8n workflow #04)

**Core Value**: Upgrades from "detecting whether there is a reply" to "understanding reply intent and automatically advancing the pipeline".

**Reply Intent Classification** (Gemini determines):

| Intent Type | Example Reply | Automatic Action |
|---------|---------|---------|
| `INTERESTED` | "Very interested, when can we chat?" | Auto-create calendar invite + mark HOT |
| `NOT_NOW` | "Budget is tight right now, let's talk next quarter" | Mark for follow-up in 90 days |
| `REFERRAL` | "Please contact our IT department's John" | Extract new contact and add to pipeline |
| `OOO` | "I'm out of office until 3/15" | Pause sequence, resume after the date |
| `UNSUBSCRIBE` | "Please remove me from your list" | Immediately stop all follow-ups, handle compliantly |
| `NEGATIVE` | "Not interested" | Mark CLOSED_LOST |

**n8n Implementation**:
```
Webhook(reply received) → HTTP(call Gemini for classification) → Switch(branch by intent) → branch actions
```
